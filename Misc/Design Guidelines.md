**Design Guidelines - Election Intelligence Platform**

**1\. Design Vision**

The platform should feel:

- intelligent
- trustworthy
- modern
- simple
- data-driven
- easy to understand for non-technical users

The UI should communicate:

"complex election intelligence made simple."

Avoid:

- cluttered dashboards
- excessive charts
- overly political visual identity
- spreadsheet-style interfaces

**2\. Design Principles**

**A. Simplicity First**

Election data is already complex.

The interface must:

- reduce cognitive load
- prioritize clarity
- guide attention intentionally

**Rule**

One screen = one primary purpose.

**B. Insight Over Raw Data**

Do not overwhelm users with numbers.

Instead:

- summarize
- highlight anomalies
- explain trends
- prioritize key findings

Good UI:

"Party A gained momentum in coastal districts."

Bad UI:

17 tables with raw percentages.

**C. Visual Hierarchy**

Important information must stand out immediately.

**Priority Order**

1\. AI-generated insights  
2\. Key metrics  
3\. Trend visualizations  
4\. Maps  
5\. Detailed tables

**D. Explainability**

Every visualization should answer:

- What happened?
- Why does it matter?
- What changed?

**3\. Brand Personality**

| **Attribute** | **Direction**               |
| ------------- | --------------------------- |
| Tone          | Professional                |
| Style         | Modern minimal              |
| Feel          | Analytical but approachable |
| Personality   | Intelligent and trustworthy |

Avoid:

- aggressive political branding
- sensationalism
- overly corporate enterprise styling

**4\. Color Guidelines**

**Primary Palette**

**Neutral Foundation**

Use:

- white
- slate
- zinc
- gray

These improve readability.

**Accent Colors**

Use limited accents for:

- trends
- highlights
- party representation
- warnings

**Recommended Colors**

| **Usage**       | **Color** |
| --------------- | --------- |
| Positive trends | Green     |
| Negative trends | Red       |
| Neutral data    | Blue      |
| Alerts          | Amber     |

**Important Rule**

Do NOT overload the UI with party colors.

Too many political colors create:

- visual chaos
- bias perception
- reduced readability

**5\. Typography Guidelines**

**Font Style**

**Recommended Fonts**

- Inter
- Geist
- Manrope

**Typography Hierarchy**

| **Element**     | **Size** |
| --------------- | -------- |
| Page Title      | 32-40px  |
| Section Headers | 20-24px  |
| Metric Cards    | 24-32px  |
| Body Text       | 14-16px  |
| Labels          | 12-14px  |

**Typography Principles**

- high contrast
- generous spacing
- avoid dense paragraphs
- emphasize insights over tables

**6\. Layout Guidelines**

**Grid System**

Use:

- 12-column responsive grid
- consistent spacing
- modular cards

**Spacing Rules**

| **Element**       | **Spacing** |
| ----------------- | ----------- |
| Section padding   | 24-40px     |
| Card padding      | 16-24px     |
| Gap between cards | 16-24px     |

**Responsive Design**

System must work across:

- desktop
- tablet
- mobile

**7\. Dashboard Design Guidelines**

**Dashboard Philosophy**

Dashboard should answer:

"What is happening right now?"

within 5 seconds.

**Recommended Layout**

\------------------------------------------------  
Top Navigation  
\------------------------------------------------  
<br/>AI Insight Banner  
<br/>Summary Cards Row  
<br/>Main Analytics Section  
\- Charts  
\- Maps  
\- Trends  
<br/>Constituency Analysis  
<br/>Detailed Data Tables

**8\. Card Design Guidelines**

**Card Structure**

Each card should contain:

**A. Title**

Clear and concise.

Example:

- "Turnout Increase"
- "Closest Contests"

**B. Primary Metric**

Large numeric focus.

**C. Supporting Insight**

Simple explanation.

Example:

"Urban turnout increased 11%."

**D. Visual Cue**

Optional:

- mini chart
- trend arrow
- heat indicator

**9\. Data Visualization Guidelines**

**General Rules**

**Prioritize Readability**

Avoid:

- excessive animations
- 3D charts
- crowded graphs

**Recommended Chart Types**

| **Use Case**           | **Chart**  |
| ---------------------- | ---------- |
| Vote share comparison  | Bar chart  |
| Historical trends      | Line chart |
| Seat distribution      | Donut/Pie  |
| Turnout analysis       | Area chart |
| Constituency intensity | Heatmap    |

**Chart Rules**

**Keep Labels Visible**

Do not overload axes.

**Limit Colors**

Maximum:

- 4-6 chart colors

**Use Consistent Scales**

Prevent misleading interpretation.

**10\. Map Design Guidelines**

**Objectives**

Maps should:

- simplify regional analysis
- highlight political shifts
- support drill-down exploration

**Recommended Features**

**Interactive Hover**

Show:

- constituency
- leading candidate
- turnout
- margin

**Zoom Support**

Enable state/district exploration.

**Color Usage**

Use:

- subtle political coloring
- opacity for turnout intensity
- hover highlighting

Avoid:

- neon colors
- oversaturated maps

**11\. AI Insight Design Guidelines**

**Most Important UI Element**

AI insights should be the centerpiece.

**Insight Card Structure**

**Insight Title**

Example:

"Turnout Surge Detected"

**Explanation**

Example:

"Turnout increased 14% in urban constituencies compared to the previous election."

**Impact Statement**

Example:

"This trend historically benefits Party A."

**Tone Guidelines**

Insights must:

- remain neutral
- avoid political bias
- explain clearly
- avoid sensational language

**12\. Natural Language Query UI**

**Design Goal**

Users should feel like:

"They are talking to election intelligence."

**Chat UI Guidelines**

**Recommended Features**

- suggested questions
- conversation history
- quick filters
- answer citations

**Response Style**

Responses should:

- be concise
- explain findings
- reference charts/maps when possible

**13\. Navigation Guidelines**

**Navigation Structure**

**Primary Navigation**

| **Section**    | **Purpose**               |
| -------------- | ------------------------- |
| Dashboard      | Overview                  |
| Constituencies | Drill-down analysis       |
| Insights       | AI-generated findings     |
| Maps           | Geographic visualization  |
| Ask AI         | Natural language querying |

**Navigation Rules**

- shallow hierarchy
- max 2 clicks to important data
- sticky top navigation preferred

**14\. Accessibility Guidelines**

**Requirements**

**Color Accessibility**

Ensure contrast compliance.

**Typography Accessibility**

Minimum:

- 14px readable text

**Keyboard Accessibility**

Support:

- tab navigation
- focus indicators

**Screen Reader Support**

Use semantic HTML.

**15\. Motion & Animation Guidelines**

**Animation Philosophy**

Subtle and purposeful.

**Recommended Usage**

Use animation for:

- chart transitions
- hover states
- loading indicators
- map interactions

**Avoid**

- excessive motion
- flashy effects
- distracting transitions

**16\. Loading & Empty States**

**Loading States**

Use:

- skeleton loaders
- progressive rendering

**Empty States**

Provide:

- helpful instructions
- upload prompts
- example datasets

**17\. Error Handling Guidelines**

**Error Messaging**

Errors must:

- explain clearly
- suggest resolution
- avoid technical jargon

Example:

"Invalid CSV format. Please upload a file with constituency and vote columns."

**18\. Mobile Design Guidelines**

**Mobile Priorities**

On mobile:

- prioritize insights first
- collapse secondary charts
- simplify maps

**Mobile Layout**

Insights  
↓  
Summary Cards  
↓  
Charts  
↓  
Maps  
↓  
Tables

**19\. Recommended Component Library**

**UI Stack**

| **Purpose**   | **Technology**     |
| ------------- | ------------------ |
| UI Components | shadcn/ui          |
| Styling       | Tailwind CSS       |
| Icons         | Lucide             |
| Charts        | Recharts / ECharts |
| Maps          | Leaflet / Mapbox   |

**20\. Design Anti-Patterns to Avoid**

**Avoid These**

❌ Too many charts  
❌ Spreadsheet-like UI  
❌ Political propaganda styling  
❌ Overuse of party colors  
❌ Tiny unreadable labels  
❌ Heavy animations  
❌ Dense dashboards  
❌ Excessive filters

**21\. Ideal User Experience**

**User Journey**

**Step 1**

Upload election dataset.

**Step 2**

Receive instant AI-generated summary.

**Step 3**

Explore:

- trends
- maps
- turnout
- party performance

**Step 4**

Ask questions naturally.

**22\. Final Design Goal**

The final experience should feel like:

"A modern AI-powered election intelligence terminal that transforms complex electoral data into understandable, actionable insights for everyone."
