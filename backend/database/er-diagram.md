## Key Design Decisions

1. **Many-to-Many Relationship**: Elections вҶ” Constituencies handled via junction table `election_constituencies`
2. **Historical Versioning**: Results table includes `version` and `is_final` columns for tracking changes
3. **Soft Deletes**: Users, elections, constituencies, and candidates have `deleted_at` for soft deletion
4. **JSONB Columns**: Datasets, AI insights, and reports use JSONB for flexible metadata storage
5. **Timestamp Tracking**: All tables have `created_at`, and relevant tables have `updated_at` with triggers
6. **Foreign Key Cascades**: Appropriate CASCADE rules to maintain referential integrity
